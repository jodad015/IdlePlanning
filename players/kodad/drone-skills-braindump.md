# Drone Skills & Chassis Brain Dump

## The Chassis Model

You don't just unlock a skill — you unlock a **chassis type**. Each chassis is purpose-built for a category of work. Once you have a chassis, you can build, deploy, and upgrade drones of that type.

This gives progression two layers:
1. **Unlock the chassis** (big milestone, new category of work)
2. **Build/deploy/upgrade drones within that chassis** (ongoing progression within the skill)

## Active Play: Mini-Games Per Skill

Each skill has an **active mini-game** — something where you're actually touching and interacting with the screen. This is the active play layer on top of the idle progression. All games should be playable **one-thumb in portrait mode on a phone**.

### Mini-Game Ideas (from brainstorm)
- **Lawn mowing**: Drive around to hit tall patches of grass that respawn; keep mowing
- **Package delivery (quadcopter)**: Tap to release package so it lands on target, accounting for wind
- **Taxi bot**: Trace a path on a map to pick up passengers as quickly as possible
- **Fishing**: Quick reaction challenge to reel in fish / time the catch
- **Hunting**: Track and react to prey — timing/reflex based

### Complexity Progression
- Simple drones = simple mini-games (lawn mowing navigation, package drop timing)
- Complex drones = multi-system mini-games (warehouse management, corporate espionage with obstacles)
- Both the real-world complexity of the bot AND the mini-game complexity should scale together
- Early mini-games teach core mechanics; later ones layer on what you've learned
- Each new skill unlock should feel like a genuinely different game to play — re-engages the player
- If a skill takes a month to unlock, the mini-game needs to be tempting enough to justify the wait

### Granularity Question
- Should one drone handle multiple sub-tasks (landscaping drone does mowing AND trimming)?
- Or are those separate drones (lawn specialist vs. arborist)?
- Tension: too granular = overwhelming at start; too broad = fewer milestone moments
- **Still undecided** — need to find the right balance of novelty vs. complexity
